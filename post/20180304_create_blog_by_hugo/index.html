使用Hugo和GitHub Pages搭建个人博客

2018-03-02 约 800 字 预计阅读 2 分钟

安装Hugo

用brew命令安装hugo。

$ brew install hugo

创建站点

使用如下命令在当前目录下创建站点框架。

$ hugo new site blog
$ tree blog
$ tree blog
blog
├── archetypes
│   └── default.md
├── config.toml
├── content
├── data
├── layouts
├── static
└── themes

6 directories, 2 files

使用git管理blog代码。

$ cd blog
$ git init
$ git add .
$ git commit -m 'init commit'

安装主题

在Hugo Themes里选择主题，以主题Even为例，使用submodule的方式安装到blog/themes文件夹下。

$ git submodule add https://github.com/olOwOlo/hugo-theme-even.git themes/even
$ git add .
$ git commit -m 'add themes/even as submodule'

根据主题文档说明，编辑config.toml文件配置主题。

$ vi config.toml
$ git add .
$ git commit -m 'config for even theme'

创建文章

使用hugo new post/first.md命令，会生成content/post/first.md文件，其初始内容如下。

---
title: "First"
date: 2018-03-02T08:39:27+09:00
draft: true
---

在两段---之间的内容被称为Front Matter，用它作为元数据来控制每篇文章，例如文章的标题title，是否为草稿draft，标签tags，分类categories等。

文章内容以markdown格式写在front matter之后，例如

---
title: "第一篇博客"
date: 2018-03-02T08:39:27+09:00
draft: true
---
用hugo生成的第一篇博客。

使用hugo server命令启动本地服务来预览博客内容。

$ hugo server --buildDrafts

打开浏览器，访问http://localhost:1313预览文章内容，确认无误后，编辑文章front matter，设置draft: true。

部署到GitHub Pages

配置好GitHub Pages的repo后，将其作为submodule加到public文件夹下。public也是hugo生成所有博客静态文件所在的目录。

 git submodule add git@github.com:wangyuehong/wangyuehong.github.io.git public

执行hugo命令，生成静态文件到public目录。

$ hugo

将public目录下的内容部署到GitHub Pages。

$ cd public
$ git add .
$ git commit -m "rebuild site"
$ git push origin master

最后

实际效果就是这样，blog的repo，GitHub Pages的repo，实际站点就是现在看到的老王备忘录。
